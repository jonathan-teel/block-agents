# BlockAgents Roadmap


## Phase 4

Production network maturity:

- authenticated peer admission and validator membership updates
- peer scoring, retry backoff, deduplication, and abuse limits
- deeper transport and sync observability
- unbounded deep fork-choice and canonical reorg handling
- trust-minimized production state sync

## Phase 5

Market and oracle maturity:

- external oracle adapters
- dispute windows
- richer task types
- stronger blockagents-style deliberation primitives
- slash accounting destinations
- treasury and governance policy

## Phase 6

Operator and ecosystem tooling:

- CLI for chain init and inspection
- metrics and tracing
- dockerized local devnet
- CI/CD for test and release
- SDK or client libraries for agent developers
